# Plugin Development Guide

Extend Fost with custom plugins for validation, code generation, and transformations.

## Overview

Plugins allow you to:
- Add custom validation rules
- Extend code generation
- Transform specifications before generation
- Add custom code formatters
- Integrate with external services
- Create custom test generators

## Plugin Architecture

### Plugin Interface

```typescript
export interface IPlugin {
  name: string;
  version: string;
  enabled: boolean;
  
  hooks: {
    onValidate?: (context: ValidationContext) => Promise<ValidationResult>;
    onGenerate?: (context: GenerationContext) => Promise<void>;
    onTransform?: (spec: any) => Promise<any>;
    onFormat?: (code: string) => Promise<string>;
    onTest?: (context: TestContext) => Promise<TestResult>;
  };
  
  config?: {
    [key: string]: any;
  };
}
```

## Getting Started

### Create a Plugin

```typescript
// plugins/my-custom-plugin.ts
import { IPlugin, ValidationContext, ValidationResult } from 'fost';

export class MyCustomPlugin implements IPlugin {
  name = 'my-custom-plugin';
  version = '1.0.0';
  enabled = true;
  
  config = {
    strictMode: true,
    customRules: []
  };

  constructor(options?: any) {
    if (options) {
      this.config = { ...this.config, ...options };
    }
  }

  async onValidate(context: ValidationContext): Promise<ValidationResult> {
    const errors: any[] = [];
    
    // Custom validation logic
    if (this.config.strictMode) {
      // Add your validation rules
      if (!context.spec.info?.title) {
        errors.push({
          code: 'MISSING_TITLE',
          message: 'API specification must have a title',
          suggestion: 'Add "title" to info section'
        });
      }
    }
    
    return {
      valid: errors.length === 0,
      errors
    };
  }
}
```

### Register Plugin

**In configuration:**

```json
{
  "fost": {
    "plugins": [
      {
        "name": "my-custom-plugin",
        "enabled": true,
        "path": "./plugins/my-custom-plugin.ts",
        "options": {
          "strictMode": true,
          "customRules": ["rule1", "rule2"]
        }
      }
    ]
  }
}
```

**Or programmatically:**

```typescript
import { FostAPI } from 'fost';
import { MyCustomPlugin } from './plugins/my-custom-plugin';

const api = new FostAPI();
api.registerPlugin(new MyCustomPlugin({ strictMode: true }));
```

## Plugin Hooks

### onValidate Hook

Validate specifications before generation.

```typescript
async onValidate(context: ValidationContext): Promise<ValidationResult> {
  const { spec, type, config } = context;
  
  const errors: any[] = [];
  
  // Check for required fields
  if (type === 'web2' && !spec.info?.title) {
    errors.push({
      code: 'NO_TITLE',
      message: 'API must have a title',
      location: 'info.title',
      suggestion: 'Add a title to the info section'
    });
  }
  
  return { valid: errors.length === 0, errors };
}
```

### onTransform Hook

Modify specification before generation (e.g., add defaults, rewrite paths).

```typescript
async onTransform(spec: any): Promise<any> {
  // Clone the spec
  const transformed = JSON.parse(JSON.stringify(spec));
  
  // Add x-internal paths to internal tag
  if (transformed.paths) {
    Object.entries(transformed.paths).forEach(([path, pathItem]: any) => {
      if (path.startsWith('/internal')) {
        pathItem['x-internal'] = true;
      }
    });
  }
  
  return transformed;
}
```

### onGenerate Hook

Modify generation process (e.g., add headers, inject custom code).

```typescript
async onGenerate(context: GenerationContext): Promise<void> {
  const { outputPath, config, analysis } = context;
  const fs = require('fs/promises');
  
  // Add custom header comment
  const indexPath = `${outputPath}/index.ts`;
  const content = await fs.readFile(indexPath, 'utf-8');
  
  const header = `/**
 * Generated by Fost with ${this.name}
 * Generated at: ${new Date().toISOString()}
 */\n\n`;
  
  await fs.writeFile(indexPath, header + content);
}
```

### onFormat Hook

Custom code formatting (before/after linting).

```typescript
async onFormat(code: string): Promise<string> {
  // Custom formatting logic
  // Example: ensure consistent spacing
  return code
    .split('\n')
    .map(line => line.trimRight())
    .join('\n');
}
```

### onTest Hook

Generate or customize tests.

```typescript
async onTest(context: TestContext): Promise<TestResult> {
  const { sdkPath, type, config } = context;
  
  // Generate custom test suite
  const tests = [];
  
  if (type === 'web2') {
    tests.push({
      name: 'should validate headers',
      assertions: [/* ... */]
    });
  }
  
  return {
    totalTests: tests.length,
    passedTests: tests.length,
    failedTests: 0
  };
}
```

## Real-World Examples

### Example 1: API Maturity Validator

Validate API spec against maturity level requirements.

```typescript
export class MaturityPlugin implements IPlugin {
  name = 'api-maturity-validator';
  version = '1.0.0';
  enabled = true;
  
  config = {
    minLevel: 'beta' // 'alpha', 'beta', 'stable'
  };

  async onValidate(context: ValidationContext): Promise<ValidationResult> {
    const { spec } = context;
    const errors: any[] = [];
    
    const maturityLevel = spec['x-maturity-level'] || 'alpha';
    const levels = ['alpha', 'beta', 'stable'];
    const currentIndex = levels.indexOf(maturityLevel);
    const minIndex = levels.indexOf(this.config.minLevel);
    
    if (currentIndex < minIndex) {
      errors.push({
        code: 'MATURITY_TOO_LOW',
        message: `API maturity (${maturityLevel}) below minimum (${this.config.minLevel})`,
        suggestion: `Set x-maturity-level: "${this.config.minLevel}"`
      });
    }
    
    return { valid: errors.length === 0, errors };
  }
}
```

### Example 2: Deprecation Warning Plugin

Add warnings for deprecated endpoints.

```typescript
export class DeprecationPlugin implements IPlugin {
  name = 'deprecation-notification';
  version = '1.0.0';
  enabled = true;

  async onGenerate(context: GenerationContext): Promise<void> {
    const { analysis, outputPath } = context;
    const fs = require('fs/promises');
    
    // Find deprecated methods
    const deprecated = analysis.methods.filter((m: any) => m.deprecated);
    
    if (deprecated.length > 0) {
      const warnings = `
/**
 * DEPRECATION WARNINGS:
 * The following methods are deprecated and will be removed in v2.0.0:
 ${deprecated.map((m: any) => ` * - ${m.name}: ${m.deprecationMessage}`).join('\n')}
 */
`;
      
      const warningsPath = `${outputPath}/DEPRECATIONS.md`;
      await fs.writeFile(warningsPath, warnings);
    }
  }
}
```

### Example 3: Security Scanner Plugin

Scan for security issues in API specifications.

```typescript
export class SecurityScannerPlugin implements IPlugin {
  name = 'security-scanner';
  version = '1.0.0';
  enabled = true;
  
  config = {
    enforceHTTPS: true,
    requireAuth: true,
    checkRateLimit: true
  };

  async onValidate(context: ValidationContext): Promise<ValidationResult> {
    const { spec } = context;
    const errors: any[] = [];
    
    // Check HTTPS
    if (this.config.enforceHTTPS && spec.servers) {
      spec.servers.forEach((server: any, idx: number) => {
        if (!server.url.startsWith('https://')) {
          errors.push({
            code: 'INSECURE_PROTOCOL',
            message: `Server ${idx} uses HTTP instead of HTTPS`,
            location: `servers[${idx}].url`,
            suggestion: 'Use HTTPS URLs for security'
          });
        }
      });
    }
    
    // Check authentication
    if (this.config.requireAuth) {
      const hasAuth = spec.securitySchemes && Object.keys(spec.securitySchemes).length > 0;
      if (!hasAuth) {
        errors.push({
          code: 'NO_AUTH',
          message: 'API has no authentication methods defined',
          location: 'components.securitySchemes',
          suggestion: 'Add API key, OAuth 2.0, or other security scheme'
        });
      }
    }
    
    return { valid: errors.length === 0, errors };
  }
}
```

### Example 4: Web3 Contract Analyzer

Analyze smart contract ABIs for best practices.

```typescript
export class Web3AnalyzerPlugin implements IPlugin {
  name = 'web3-contract-analyzer';
  version = '1.0.0';
  enabled = true;

  async onValidate(context: ValidationContext): Promise<ValidationResult> {
    const { spec, type } = context;
    const errors: any[] = [];
    
    if (type !== 'web3') return { valid: true, errors: [] };
    
    // Check for required functions
    const requiredFunctions = ['constructor'];
    const functionNames = spec.map((item: any) => item.name);
    
    // Check for event definitions
    const hasEvents = spec.some((item: any) => item.type === 'event');
    if (!hasEvents) {
      errors.push({
        code: 'NO_EVENTS',
        message: 'Contract has no event definitions',
        suggestion: 'Add event definitions for better monitoring'
      });
    }
    
    // Check for fallback function
    const hasFallback = spec.some((item: any) => item.type === 'fallback');
    
    return { valid: errors.length === 0, errors };
  }
}
```

## Testing Plugins

```typescript
import { describe, it, expect } from 'vitest';
import { MyCustomPlugin } from './my-custom-plugin';

describe('MyCustomPlugin', () => {
  it('should validate required title', async () => {
    const plugin = new MyCustomPlugin();
    const result = await plugin.onValidate({
      spec: { info: {} },
      type: 'web2'
    });
    
    expect(result.valid).toBe(false);
    expect(result.errors[0].code).toBe('NO_TITLE');
  });

  it('should pass valid specs', async () => {
    const plugin = new MyCustomPlugin();
    const result = await plugin.onValidate({
      spec: { info: { title: 'My API' } },
      type: 'web2'
    });
    
    expect(result.valid).toBe(true);
  });
});
```

## Best Practices

1. **Single Responsibility** - Each plugin should focus on one aspect (validation, transformation, etc.)
2. **Proper Error Handling** - Return meaningful error codes and suggestions
3. **Configuration** - Make plugins configurable for different use cases
4. **Testing** - Test plugins thoroughly with various specifications
5. **Documentation** - Document plugin functionality and options
6. **Performance** - Optimize for large specifications
7. **Logging** - Provide debug information when requested
